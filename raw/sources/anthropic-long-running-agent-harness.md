## Тезисы блога (по вторичным пересказам, помечено как "не проверено напрямую")
- Проблема: агенты всё чаще берут задачи на часы/дни работы, но каждая новая сессия начинается без памяти о предыдущей — согласованный прогресс через границы контекстных окон остаётся открытой проблемой.
- Anthropic предложил **три-агентный harness** для full-stack/frontend разработки: разделение на **planning / generation / evaluation** — разные агенты отвечают за разные роли, что помогает и с объективными задачами (тесты проходят/не проходят), и с субъективными (оценка дизайна/качества).
- Ключевой инсайт: агент с чистым (свежим) контекстным окном должен уметь быстро понять состояние работы — решается через файл **`claude-progress.txt`** (обновляемый лог прогресса) вместе с git-историей как источником истины.

## README репозитория `anthropics/cwc-long-running-agents` (прочитано напрямую)

### Назначение
Демонстрирует **три базовых примитива** для построения долгоживущих Claude-агентов, которые держат качество и состояние через много сессий. Основано на статьях выше, поставляется как переиспользуемые hooks + subagent-паттерн (ингредиенты, не готовый turnkey harness).

### Примитив 1 — Default-FAIL Contract (принуждение к доказательствам)
- Файлы: `hooks/track-read.sh` + `hooks/verify-gate.sh`.
- Каждая фича стартует с `"passes": false` в `test-results.json`.
- `PreToolUse`-хук не даёт агенту записать в результаты, пока он не открыл evidence (скриншоты, логи, вывод тестов) через инструмент Read.
- Эффект: агент не может заявить успех без наблюдаемого доказательства.

### Примитив 2 — Fresh-Context Evaluator (независимая оценка)
- Файл: `agents/evaluator.md` — субагент без инструментов Write/Edit.
- После каждой фичи вызывается отдельный агент, который смотрит на diff и скриншоты из контекстного окна, **никогда не видевшего сборку**. Вызов: `claude --agent evaluator -p "<review prompt>"`.
- Возвращает `PASS` или `NEEDS_WORK` с конкретными находками; при `NEEDS_WORK` находки становятся стартовым промптом следующей сессии-билдера.
- SDK-эквивалент: два отдельных вызова `query()` для генератора и оценщика — см. `evaluator_optimizer.ipynb` в `anthropics/claude-cookbooks`.

### Примитив 3 — Agent-Maintained Handoff (восстановление состояния)
- Файлы: `CLAUDE.md` + `hooks/commit-on-stop.sh`.
- Агент пишет структурированные заметки о прогрессе в `PROGRESS.md` в начале каждой сессии, коммитит осмысленную работу в git на чекпоинтах; хук `commit-on-stop.sh` подхватывает незакоммиченную работу при остановке агента. Следующая сессия перечитывает `PROGRESS.md` первым делом.
- Эффект: `git log` + структурированные progress-файлы сохраняют работу через границы контекстных окон без потери памяти при суммаризации.

### Два паттерна запуска
1. **`/goal`** (встроенный в Claude Code, без файла контракта) — например: `/goal every feature in PROGRESS.md is implemented, committed, and its tests pass`. Работает в Claude Code, headless (`claude -p`) и Remote Control; отдельная быстрая модель проверяет завершённость после каждого хода — без кастомных хуков.
2. **Свой evaluator как гейт** (с файлом контракта) — bash-цикл: пока в `test-results.json` есть `"passes": false`, запускать `claude -p "Read PROGRESS.md and build the next unfinished feature..."`, затем `claude --agent evaluator -p "Review the most recent commit..."`; при не-PASS находки уходят в `NEXT_FINDINGS.md` для следующего прохода.

### Git history & state recovery — структура
- `PROGRESS.md` — заметки агента (фичи, статус, находки).
- Git commits на каждом чекпоинте.
- `commit-on-stop.sh` — бэкстоп-хук, ловит незакоммиченное при остановке.
- Наблюдаемость: `watch` на `PROGRESS.md`, `git log --oneline`, папку скриншотов, счётчик evidence-reads.

### Структура `.claude/` в репозитории
```
.claude/
├── hooks/
│   ├── track-read.sh          # логирует evidence-чтения
│   ├── verify-gate.sh         # блокирует запись без evidence
│   ├── commit-on-stop.sh      # коммитит незавершённую работу при остановке
│   ├── kill-switch.sh         # останов через файл AGENT_STOP
│   └── steer.sh                # перенаправление через файл STEER.md
├── agents/
│   └── evaluator.md           # субагент-оценщик без Write/Edit
└── CLAUDE.md                  # системный промпт + инструкции по handoff
```
Операторские контролы: `kill-switch.sh` (тронуть файл `AGENT_STOP` — остановить все tool calls), `steer.sh` (записать в `STEER.md` — перенаправить агента на ходу без рестарта).

### Не входит в репозиторий ("Going Further")
Planner-агент (разворачивает однострочную задачу в `BUILD_PLAN.md`), sprint-контракты (файл с определением "done" на фичу), рубрики оценки субъективного качества, evaluator, открывающий реальное приложение в браузере вместо доверия скриншотам, хостед-раннтайм (Claude Managed Agents).

### Итоговый тезис репозитория
"This repo ships example ingredients, not a turnkey harness" — примитивы независимы друг от друга, каждый переносится напрямую в `PreToolUse`/`Stop`-колбэки Claude Agent SDK.
