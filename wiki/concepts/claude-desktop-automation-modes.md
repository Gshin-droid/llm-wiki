# Автоматизации Claude Desktop (Scheduler, Routines, /loop)

## Определение
Четыре способа заставить Claude работать без прямого участия пользователя, доступные через приложение **Claude Desktop** (три вкладки: Chat, Cowork, Code), описанные в [[nikita-efimov-claude-automations]]. Дополняет [[claude-cowork]] (там были только упомянуты вскользь) конкретной механикой и ограничениями каждого способа.

## Четыре способа, от простого к сложному

### 1. Cowork Scheduler
Задача формулируется обычными словами ("каждый день в 21:00 проверяй папку загрузок..."). Требует коннектор **Desktop Commander** для доступа к файлам/папкам на компьютере. Работает, только пока компьютер включён и приложение открыто.

### 2. Local Routines (вкладка Code)
Механически то же самое, что Scheduler, но с возможностью подключиться к той же сессии из терминала/Claude Code.

### 3. Cloud Routines (remote)
Выполняются на серверах Anthropic независимо от состояния компьютера пользователя. Ограничения: минимальный интервал — 1 час (чаще — только Local); лимит запусков/сутки по тарифу (Pro — 5, Max — 15, Team — 25); нет доступа к локальным файлам — только к явно подключённым облачным сервисам (почта, календарь, Google Docs, БД). Три типа триггеров, комбинируемых в одной автоматизации: по расписанию, по вебхуку (входящий запрос от внешнего сервиса), по API — что превращает автоматизацию из таймера в реактивного агента.

### 4. `/loop`
Короткий цикл внутри текущей активной сессии (не новая сессия каждый раз). Имеет доступ ко всему контексту разговора и подключённым в сессии инструментам. Живёт, пока сессия открыта, максимум 7 дней.

## Общие технические нюансы (Scheduler/Routines)
- Каждый запуск — чистая новая сессия без памяти о предыдущих.
- **Catch-up**: если компьютер был выключен в момент срабатывания, при включении подтягивается только последний пропущенный запуск за 7 дней, а не все — частые задачи (ежечасные) на время длительного отсутствия лучше ставить на паузу.
- Обязательная практика перед постановкой на расписание: несколько ручных запусков (Run Now) с правкой инструкции, пока результат не станет стабильно нужным — иначе автоматизация тихо льёт кривые результаты незамеченной.

## Как выбирать способ
Задача трогает локальные файлы → Cowork Scheduler / Local Routines. Задача про облачные сервисы и должна работать в отпуске/при выключенном компьютере → Cloud Routines. Короткий цикл на ближайшие часы в рамках текущей работы → `/loop`.

## Главный риск (по автору источника)
Автономность требует явно прописанной инструкции — агент не "читает мысли", а действует по написанному; расплывчатая задача превращается в лишний расход токенов и риск нежелательных действий, а не в экономию времени.
