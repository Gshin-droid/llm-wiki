# MCP (Model Context Protocol)

**Актуально на:** 2026-07-10

## Что это
Открытый протокол Anthropic для подключения LLM-агентов к внешним системам — базам данных, API, браузерам, файловым хранилищам, SaaS-сервисам (Notion, Slack, Stripe, GitHub, X и сотни других) — единым стандартным способом вместо кастомной интеграции под каждый сервис. Работает поверх JSON-RPC. В [[claude-code]] и [[claude-agent-sdk]] MCP-серверы — один из основных способов расширить возможности агента за пределы встроенных инструментов (Read/Bash/Grep и т.п.).

До этой страницы MCP в вики упоминался только вскользь ("мост к внешним сервисам" в разделе claude-code.md, способ подключения Obsidian) — источник [[mcp-2026-07-28-spec-release-candidate]] стал поводом развернуть его в отдельный концепт.

## Архитектура (на 2026-07-09)
- Модель клиент-сервер: **host** (приложение, например Claude Code) держит одного или нескольких **клиентов**, каждый клиент подключён к одному **MCP-серверу**.
- Сервер объявляет капабилити (tools, resources, prompts) и версию протокола.
- До версии `2026-07-28`: подключение начиналось с handshake (`initialize`/`initialized`), сервер выдавал `Mcp-Session-Id`, и клиент "прилипал" к конкретному инстансу сервера (stateful).

## Крупнейшая ревизия — спецификация 2026-07-28
RC вышел 21 мая 2026, финальная версия **запланирована на 28 июля 2026 и на момент обновления страницы ещё не вышла** — `2026-07-28` в названии это идентификатор версии спецификации (MCP версионирует спеки датой планового релиза). Это **самая крупная ревизия протокола с момента запуска**, с breaking changes и 10-недельным окном валидации для SDK. _(Исправлено 2026-07-10: ранее страница ошибочно утверждала, что финал уже вышел.)_ Ключевые изменения — подробности в [[mcp-2026-07-28-spec-release-candidate]]:
- **Протокол становится stateless** — убран handshake и `Mcp-Session-Id`, метаданные клиента едут в `_meta` на каждом запросе. Любой запрос обслуживается любым инстансом сервера → масштабирование за обычным round-robin балансировщиком без sticky-сессий. Приложениям, которым всё же нужно состояние, доступен паттерн "handle" (сервер возвращает идентификатор, модель передаёт его в следующих вызовах).
- **MCP Apps** (расширение) — server-rendered интерактивные HTML-интерфейсы в sandboxed iframe, с тем же аудиторским путём, что у обычных tool calls.
- **Tasks** (было core-фичей → стало расширением) — асинхронные долгие вызовы: сервер сразу возвращает хендл задачи, клиент опрашивает `tasks/get`/`tasks/update`/`tasks/cancel`.
- Авторизация приближена к OAuth 2.0/OpenID Connect (валидация `iss`, привязка credentials к authorization-серверу).
- Формальная политика депрекации (Roots, Sampling, Logging депрекируются, минимум 12 месяцев до возможного удаления) — протокол впервые формализовал, как безопасно убирать старые фичи.
- Enterprise-Managed Authorisation (централизованная авторизация через IdP организации) продвинута до stable, принята Anthropic, Microsoft, Okta (источник: InfoQ, контекст к тому же поиску, не основной источник).

## Как используется в этой вики / у пользователя
- [[claude-code]]: MCP-серверы — мост к внешним сервисам прямо из диалога; `claude mcp login/logout` (появилось в версии 2.1.186, см. [[claude-code-changelog-snapshot-2026-07]]) — CLI-аутентификация без интерактивного меню.
- [[claude-agent-sdk]]: MCP — одна из шести капабилити SDK "из коробки" (`mcp_servers` опция), пример из официальной документации — подключение Playwright MCP для браузерной автоматизации.
- [[obsidian]]: один из двух способов подключить Obsidian к Claude Code — через MCP-сервер (второй — прямой файловый доступ).
- [[maxim-bashkardinov]] / [[bashkardinov-obsidian-claude-guide]]: гайд именно про MCP-подключение Obsidian для документации кодового проекта.
