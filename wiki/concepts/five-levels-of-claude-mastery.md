# Пять уровней владения Claude

## Определение
Модель зрелости использования Claude из [[ai-proryv-5-levels-claude]]: переход между уровнями определяется не знанием технологий, а готовностью передавать контроль системе.

## Уровни
1. **Энтузиаст** — Claude как поисковик. Вопрос → ответ, каждый чат с нуля. Лайфхак: прикреплять скриншоты вместо долгих текстовых описаний. Шаг вперёд: завести первый [[claude-projects|Project]] с базовым описанием "кто я и что важно учитывать".
2. **Новичок** — память и поиск по чатам внутри проекта, [[claude-projects|коннекторы]] (Google Drive, Slack, GitHub, Notion), создание реальных файлов (Excel/Word/PowerPoint, не описаний), артефакты (мини-приложения в чате), графики из CSV. Шаг вперёд: перейти на десктопное приложение и [[claude-cowork|Cowork]].
3. **Средний** — доступ к файловой системе компьютера, Computer Use (работа с интерфейсами без готовой интеграции), [[claude-skills|Skills]], управление с телефона (Dispatch). Шаг вперёд: завести файл `about-me`/`CLAUDE.md` с правилами проекта.
4. **Продвинутый** — работа через терминал ([[claude-code]]), файл `CLAUDE.md` как накопитель правил из реальных ошибок, режим планирования (Opus планирует → Sonnet исполняет), параллельные субагенты, самопроверка (агент сам тестирует результат до показа пользователю). Барьер: пользователь становится узким местом, переключаясь между агентами.
5. **Архитектор** — облачные рутины по расписанию/событию (не зависят от включённого компьютера), хуки (логика до/после действия агента), управление через мессенджер, Autodream (фоновая очистка/разрешение противоречий в памяти между сессиями), команды агентов (предлагает/критикует/улучшает по кругу), task budgets (лимит ресурсов на автономную задачу). Главный барьер — не технический, а доверие: строится итеративно через "пустую парковку" (начинать с безобидных задач и постепенно расширять автономность).
