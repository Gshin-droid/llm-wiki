# Контент-ферма через ИИ-агента

## Определение
Техника производства монетизируемого контента (статьи для доход-платформ типа Яндекс.Дзен, посты для соцсетей) в промышленных масштабах через агента с доступом к файлам и браузеру, описанная в [[woome-ai-dzen-content-automation]].

## Пайплайн
1. **Разведка ниши** — выбор темы по данным о доходности каналов на самой платформе (не вслепую).
2. **Анализ конкурентов агентом-скрапером** — агент с доступом к браузеру заходит на канал конкурента, выгружает заголовки+просмотры за N дней в структурированный файл. Заменяет ручной сбор данных.
3. **"Система" вместо промта** — по результатам разведки собирается не один запрос, а комплект файлов (инструкция, стиль, структура, стоп-лист штампов/ошибок нейросети по нише). Тот же принцип, что в [[skill-authoring-practical-rules]]: composability вместо монолитного промта.
4. **Параллельная генерация** — N независимых потоков агента пишут N единиц контента одновременно по общей "системе", без координации между собой (проще, чем оркестрация в [[agent-teams]] — не нужен обмен сообщениями между заданиями, задачи полностью независимы).
5. **Фильтр качества перед публикацией** — специфичный для платформ с антифрод-политикой шаг: прогон текста через AI-детектор перед выкладыванием.

## Наблюдение
Раскрывает практический паттерн "система из файлов вместо промта + параллельные независимые потоки" на новом домене (контент-производство, а не кодинг) — тот же принцип модульности скиллов уже встречался в кодинге ([[skill-authoring-practical-rules]]) и в разработке приложений ([[vibecoding-full-workflow]]).

## Оговорка
Источник — маркетинговое видео с продажей платного комплекта материалов; финансовые цифры дохода не проверяемы, зафиксирован только техническая часть метода.
