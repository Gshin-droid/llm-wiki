# "Второй мозг": ежедневные воркфлоу Obsidian + Claude Code

## Определение
Из [[sonny-huynh-second-brain]]: более лёгкий, продуктивностный (не knowledge-base) паттерн — три плагина (Tasks, Templater, Dataview) + Claude Code для рутинных ежедневных операций с заметками.

## Три воркфлоу
1. **Утренний брейншторм** — Claude генерирует ежедневную заметку по шаблону, подтягивает незакрытые задачи со вчера, связывает с текущими проектами (15 минут копипаста → 30 секунд)
2. **Обработка заметок со встреч** — сырые заметки → Claude извлекает пункты действий, форматирует как Obsidian Tasks с датами на основе контекста обсуждения
3. **Еженедельный обзор** — саммари недели по ежедневным заметкам, выполненные задачи, заблокированные проекты, приоритеты на следующую неделю

## Отличие от паттерна этой вики
Не про накопление знаний из внешних источников ([[persistent-wiki-pattern]]), а про личную продуктивность/менеджмент задач поверх заметок, которые пишет сам пользователь. Dataview-дашборд (канбан задач, стрик привычек) — на чистом JS внутри Dataview-блока, без внешних сервисов.

## Настройка (три плагина, без "кроличьей норы")
Автор намеренно ограничивается тремя community-плагинами Obsidian вместо полного стека продуктивности:
- **Tasks** — задачи в формате `- [ ] текст 📅 дата #тег` в любом месте хранилища, с запросами (`tasks`-блоками) из любой заметки.
- **Templater** — шаблоны с JS-логикой (динамические даты, вызовы внешних API), мощнее встроенных шаблонов Obsidian.
- **Dataview** — запросы к хранилищу как к базе данных (SQL-аналогия автора), включая произвольный JS через `dataviewjs`-блоки.
Claude Code запускается прямо из корня Obsidian-хранилища (`cd ~/ObsidianVault && claude`) — доступ ко всем заметкам сразу, без дополнительной настройки MCP/коннекторов.

## Пример: Dataview-дашборд ежедневной заметки
Автор делится готовым `dataviewjs`-скриптом (вставляется в шаблон Templater ежедневной заметки), который рендерит:
- **Канбан задач по сроку**: пять колонок ("Сегодня" / "Эта неделя" / "След. неделя" / "Позже" / "Без даты"), сортировка через сравнение `task.due.ts` с границами календарной недели; чекбокс в карточке напрямую переписывает строку задачи в исходном файле через `app.vault.modify` (клик по чекбоксу = `- [ ]` → `- [x]` + дата выполнения, без открытия файла).
- **Трекер привычек-стриков**: список привычек с эмодзи-метками (🚶 шаги, 🍽️ калории, 😴 сон и т.п.), за 7 последних ежедневных заметок — сетка точек (✓/✗/—) на основе того, отмечена ли задача-привычка выполненной в заметке того дня.
- **Ярлыки папок** — кликабельные плитки-переходы в заданные папки (Work/Business/Health/Investing и т.п., задаются вручную в конфиге скрипта).
- **Виджет недавних заметок** — последние 5 изменённых файлов хранилища (кроме ежедневных заметок), сортировка по `file.mtime`.
- Статистика недели вверху (закрыто задач / открыто задач / привычки сегодня) — карточки-счётчики, посчитанные через `dv.pages().file.tasks`.
Конфигурация (папки, привычки, цвета) вынесена в начало скрипта одним блоком — под правку под свою структуру хранилища. Технику стоит иметь в виду при доработке любого будущего Obsidian-дашборда пользователя — рабочий, воспроизводимый паттерн, не только описание идеи.
