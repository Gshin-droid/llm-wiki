# Claude Artifacts

**Актуально на:** 2026-07-19

## Что это
Артефакт — живая интерактивная веб-страница, которую Claude Code (или другая поверхность Claude) публикует из сессии на приватный URL на claude.ai. Открывается в браузере и обновляется на месте по мере продолжения сессии — в отличие от статичного вывода в терминал. Используется, когда текст в терминале — не тот носитель для результата: разбор PR с аннотациями, дашборд по данным сессии, набор вариантов для сравнения бок о бок, таймлайн расследования, который заполняется по ходу долгой задачи.

Артефакт — это "снимок работы, а не приложение": одна самодостаточная страница без бэкенда, не может хранить ввод из формы и не может обслуживать несколько маршрутов. Единственный способ получить внешние данные при просмотре — вызов MCP-коннекторов (см. ниже).

## Создание, обновление, шаринг
- Claude пишет `.html` или `.md` файл в проекте и публикует его; перед первой публикацией спрашивает разрешение. Повторная публикация уже одобрённого артефакта не спрашивает снова.
- Обновление — та же публикация на тот же URL: каждая публикация становится версией, зритель может выбрать, какую версию видеть, через контрол **Share**.
- Обновить артефакт из другой сессии можно, дав Claude его URL — без URL новая сессия всегда создаёт новый артефакт.
- Новый артефакт виден только автору. Поделиться — контрол **Share** в шапке страницы:
  - **Внутри организации** (Team/Enterprise) — доступ конкретным людям или всем в организации.
  - **Публично** — ссылка без входа на claude.ai. На Pro/Max это единственный способ поделиться. На Team/Enterprise публичный шаринг по умолчанию выключен, включает Owner.
  - **Роль редактора** (Team/Enterprise) — зритель по умолчанию не может менять страницу; редактор публикует новые версии тем же способом, что обновление из другой сессии (даёт Claude URL артефакта).

## Вызов MCP-коннекторов зрителя (ключевая функция обновления 2.1.209+)
Артефакт может вызывать [[mcp-model-context-protocol|MCP-коннекторы]] при каждом открытии страницы — тогда страница показывает актуальные данные, а не снапшот момента сборки. Требует Claude Code v2.1.209+, доступно на Pro/Max/Team/Enterprise.

Механика для зрителя:
- **У каждого зрителя — свои коннекторы**: вызов идёт через подключения того аккаунта, который сейчас смотрит страницу, а не автора публикации — два человека, открывшие один дашборд, могут увидеть разные данные. Страница не видит ничьих credentials, вызовы делает сам claude.ai от её имени.
- **Зритель одобряет доступ** перед первым вызовом коннектора; отказ действует до конца этой загрузки страницы, при перезагрузке спрашивает заново.
- **Действия тоже идут от аккаунта зрителя** — если страница даёт контрол, вызывающий инструмент с побочным эффектом (написать сообщение, обновить issue), это выполняется от аккаунта того, кто нажал.
- Ответы кешируются в браузере зрителя — повторное открытие рендерится из кеша, затем обновляется свежими данными.
- Локальные MCP-серверы, настроенные в Claude Code (`.mcp.json`), могут снабжать данными на этапе сборки страницы, но опубликованная страница их вызывать не может — только коннекторы из аккаунта claude.ai зрителя, явно задекларированные при публикации.

**Важный нюанс, не попавший в исходный ingest-снапшот 2026-07-19** ([[claude-code-changelog-snapshot-2026-07-19]]): **артефакт, вызывающий коннекторы, нельзя опубликовать по публичной ссылке ни на одном тарифе** — это отдельное ограничение, не связанное с общим включением/выключением публичного шаринга. На Team/Enterprise такой артефакт можно расшарить только внутри организации или оставить приватным; на Pro/Max (где публичная ссылка — единственный способ шаринга) артефакт с коннекторами вообще остаётся приватным автору.

## Технические ограничения страницы
- Строгий CSP: блокирует запросы к любому другому хосту (скрипты, стили, шрифты, картинки, `fetch`/XHR/WebSocket) — Claude инлайнит CSS/JS и встраивает картинки как data URI. Исключение — вызовы коннекторов: их делает сам claude.ai, а не страница напрямую.
- Без бэкенда, без хранения form-ввода, без своего API.
- Одна страница — относительные ссылки на другие файлы не резолвятся, для многосекционного контента используются якоря на странице.
- Допустимые типы исходного файла: `.html`, `.htm`, `.md` (markdown рендерится как стилизованный HTML).
- Итоговый размер страницы ≤ 16 MiB.
- С v2.1.182+ Claude применяет встроенный design-skill (палитра/типографика/лейаут по умолчанию), но сначала ищет уже заданную дизайн-систему проекта (например, в `CLAUDE.md` или theme-файле) и считает её приоритетнее собственного выбора.

## Доступность
Нужны одновременно: план Pro/Max/Team/Enterprise; сессия авторизована через `/login` (не API-ключ/gateway-токен/облачные credentials); модель-провайдер — Anthropic API напрямую (недоступно на Bedrock/Vertex/Foundry); у организации не включены CMEK/HIPAA/Zero Data Retention; поверхность — Claude Code CLI v2.1.183+ или Claude Desktop v1.13576.0+ (в Agent SDK, GitHub Action и MCP-server контекстах выключено по умолчанию).

## Управление на уровне организации
Owner на Team/Enterprise включает/выключает артефакты в целом и отдельно — вызовы коннекторов из артефактов (переключатель **Enable artifact connectors**, действует и на артефакты из обычных разговоров claude.ai, не только Claude Code). Отдельные переключатели: публичный шаринг (**External sharing**, по умолчанию выключен), retention policy для приватных/расшаренных артефактов, Compliance API для листинга/чтения/удаления артефактов организации, аудит-лог событий `claude_artifact_*`.

## Связь с этой вики
Ровно та функция (вызов MCP-коннекторов зрителя при открытии страницы), которая стоит за параметром `capabilities: {mcp: ...}` инструмента `Artifact`, доступного в сессиях этой вики, и за скиллом `artifact-capabilities` (обязателен к прочтению перед объявлением capability или написанием `window.claude.mcp`-кода в артефакте). До этой страницы концепт был описан только фрагментарно, в разделе changelog-снапшота [[claude-code]] — эта страница закрывает пробел, найденный при lint-прогоне 2026-07-19 (см. `wiki/gaps-backlog.md`).
