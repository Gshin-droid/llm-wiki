# LLM Coding Guidelines (Karpathy's 4 principles)

## Определение
Набор из четырёх поведенческих принципов для снижения типичных ошибок LLM-агентов при написании кода, сформулированных на основе наблюдений [[andrej-karpathy]] и оформленных как готовый `CLAUDE.md`-файл в [[karpathy-skills-claude-md|репозитории multica-ai]].

## Проблема, которую решают
Karpathy: "Models make wrong assumptions and run with them without checking. They overcomplicate code, bloat abstractions, and sometimes change code they don't understand."

## Четыре принципа

### 1. Think Before Coding
Не молчать о допущениях и неоднозначности. Явно проговаривать предположения, представлять альтернативные интерпретации, а не выбирать одну молча. Если что-то непонятно — остановиться и спросить.

### 2. Simplicity First
Минимум кода, решающего задачу. Никаких фич "про запас", никакой спекулятивной гибкости/конфигурируемости, никакой обработки ошибок для невозможных сценариев. Тест: "сказал бы senior-инженер, что это overcomplicated?"

### 3. Surgical Changes
Трогать только то, что требуется. Не "улучшать" соседний код, не рефакторить то, что не сломано, соответствовать существующему стилю. Убирать только то, что стало мёртвым кодом из-за твоих собственных правок — не трогать существующий dead code без запроса. Тест: каждая изменённая строка должна напрямую соответствовать запросу пользователя.

### 4. Goal-Driven Execution
Превращать задачи в проверяемые критерии успеха ("Fix the bug" → "напиши тест, который воспроизводит баг, затем сделай его зелёным"). Сильные критерии успеха позволяют агенту итерироваться самостоятельно без постоянных уточнений.

## Наблюдение
Эти принципы почти дословно совпадают с базовыми инструкциями, по которым уже работает Claude Code (см. раздел "Doing tasks" системного промпта — не добавлять лишнего, хирургические правки, no error handling для невозможных сценариев). Репозиторий не открывает новую практику, а формализует и делает копируемой уже существующую — что, судя по 188.8k звёзд, оказалось очень востребовано как явный артефакт, который можно вставить в любой проект.
