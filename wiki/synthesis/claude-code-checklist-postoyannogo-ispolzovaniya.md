# Чек-лист Claude Code для постоянного использования

**Вопрос:** что из практик [[nikita-vels-claude-code-30-concepts|30 концептов Claude Code]] стоит взять на вооружение?
**Подход:** не все 30 концептов одинаково полезны прямо сейчас — ниже отфильтровано и приоритизировано под твой контекст: ведение этой вики + будущие вайбкодинг-проекты, пока без командной работы и без сложных мультиагентных систем.

## Всегда (привычка на каждый день)

- [ ] **Один диалог = одна задача.** Не тащить новую тему в старый разговор с забитым контекстом — это главная причина, почему "клод тупеет".
- [ ] **Следить за % контекста.** Больше 50% — либо `/compact` (если задача продолжается и важен контекст), либо `/new`/`/clear` (если нет).
- [ ] **Секреты — под замком с самого начала проекта.** В `settings.json`/`.claudeignore` закрывать `.env` и подобное от чтения агентом, а не полагаться на память не спалить их в коде. Прямое применение [[ai-security-by-design|минимальных привилегий]] — уже разбирали в [[pre-project-architecture-checklist]], здесь конкретный технический механизм для этого.
- [ ] **Разрешения настроены заранее, не одобрять каждые 10 секунд.** Разрешить безопасное (чтение, тесты, git, dev-сервер), держать под контролем опасное (установка пакетов, удаление файлов, внешние API-вызовы).
- [ ] **Скилы и плагины — только из проверенных источников** (Anthropic, официальный маркетплейс, verified GitHub). Уже есть explicit-правило про это в [[CLAUDE.md]] этой вики — распространить и на рабочие проекты.

## Ситуативно (осознанный выбор, не по умолчанию)

- [ ] **Модель под задачу, не "самую мощную всегда".** Sonnet — по умолчанию для 80% работы. Opus — специально для планирования архитектуры, сложного ресёрча, задач, где нужно самому придумать решение, а не выполнить готовую инструкцию. Opus Plan Mode — включить, чтобы агент сам переключался, не тратить лимиты вручную подбирая.
- [ ] **Effort high/max — для архитектуры и глубокого анализа**, low/medium — для рутины. Не оставлять на max по умолчанию, если задача простая.
- [ ] **Перед вайбкодинг-проектом — `/init`**, чтобы получить черновой `CLAUDE.md`, затем донастроить руками (архитектура, что читать первым, пошаговые планы для повторяющихся команд) — не с нуля писать самому.
- [ ] **`/resume` вместо повторного объяснения контекста**, когда возвращаешься к прерванной задаче на следующий день.

## Когда проект вырастет (не нужно прямо сейчас, но держать в голове)

- [ ] **Субагенты** — когда в одном вайбкодинг-проекте появится много независимых задач (не поместятся в один диалог без переполнения контекста). Правило источника: одна задача → один агент, несколько независимых → субагенты, сложный проект с координацией между задачами → Agent Teams.
- [ ] **Worktrees** — если понадобится вести две фичи параллельно, чтобы два диалога не затирали правки друг друга в одних файлах.
- [ ] **Автономный режим / RALPH-loop** — для большой, чётко расписанной задачи (готовый PRD/тудулист), которую можно оставить выполняться без присмотра. Не начинать с этого — сначала научиться формулировать достаточно сильные критерии успеха (см. [[llm-coding-guidelines]], принцип Goal-Driven Execution), иначе автономный агент наделает ошибок без контроля.
- [ ] **Кросс-чек через Gemini/Codex** — только для проектов с высокой ценой ошибки, где оправдана лишняя подписка и время на сравнение ответов.

## Знать, что существует (не обязательно использовать сразу)

- [ ] **Checkpoints + `/rewind`** — автоматический откат, особенно важно помнить об этом *до* включения Bypass Permissions, а не после того, как что-то удалилось.
- [ ] **Хуки** — автоматические триггеры (уведомление по завершении задачи, конвертация формата файла) без расхода токенов. Полезно один раз настроить, когда надоест вручную проверять "закончил или нет".
- [ ] **Кастомный статус-бар лимитов** — quality-of-life, не критично при текущем объёме использования; вернуться к этому, если начнёшь работать с несколькими проектами одновременно и часто упираться в лимиты.

## Осознанно не приоритет прямо сейчас
RALPH-loop, Agent Teams и кросс-чек с другими моделями — это уровень 5 по [[five-levels-of-claude-mastery]] (см. [[moy-uroven-vladeniya-claude]] — сейчас уровень 4). Не стоит хвататься за них ради самих технологий — они оправданы, когда конкретный проект реально упирается в ограничение, которое решают именно они, а не раньше.
