# Происхождение правила «конец сессии: разовая правка или в скилл навсегда?»

**Вопрос (2026-07-09):** откуда взялось правило №16 из [[moi-pravila]] и что стоит за «эффектом 30 дней».

## Цепочка происхождения

1. **Первоисточник** — доклад Barry Zhang и Mahesh Murag (создатели Agent Skills в Anthropic) «Don't Build Agents, Build Skills Instead», AI Engineer Code Summit, 25.11.2025 → [[anthropic-code-summit-build-skills-talk]]. Цитата Zhang: *«Our goal is that Claude on day 30 of working with you is going to be a lot better than Claude on day one»* — отсюда название «эффект 30 дней». Оговорка: цитата подтверждена двумя независимыми пересказами (claudehub.fr, cto4.ai), прямая расшифровка видео не читана.
2. **Вторичный источник** — PDF «HOOK» «Инженеры Anthropic не пишут промты. У них 4 правила» → [[hook-4-pravila-claude-skills]]. Их правило №4: «Обновлять каждую сессию — после решения задачи спрашивать себя, что зафиксировать в skill навсегда».
3. **Синтез в концепт** — раздел «Итеративное накопление» в [[skill-authoring-practical-rules]].
4. **Сжатие в личное правило** — п. 16 в [[moi-pravila]]. Формулировка «30 секунд» — добавлена при сжатии: цена ритуала ничтожна по сравнению с эффектом.

## Логика правила

Claude не учится сам между сессиями — каждая сессия начинается с чистого контекста. Единственный механизм накопления — файлы: зафиксированное в скилле/CLAUDE.md/памяти доживает до следующей сессии, остальное умирает с диалогом. Ритуал в конце сессии («разовый случай или повторяющийся паттерн?») — это и есть механизм, которым достигается «день 30 лучше дня 1».

Практические приёмы (из [[skill-authoring-practical-rules]]):
- Просить Claude самого предложить правки в `SKILL.md` по итогам решённой задачи — он видел, где спотыкался.
- Самый ценный накапливаемый контент — раздел **Gotchas**: не придумывается заранее, растёт только из реальных факапов. Именно он теряется без фиксации.
- Кейс из доклада: Claude каждую сессию заново писал один и тот же Python-скрипт для слайдов, пока его не вынесли в скилл.
