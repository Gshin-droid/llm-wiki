## Что нового по сравнению с уже известным
- **Формы ответа на Query не ограничены markdown-страницей** — сравнительная таблица, слайд-дек (Marp), график (matplotlib), canvas. Мы пока делали только markdown-страницы в `wiki/synthesis/` — это не ошибка, но недоиспользованная возможность.
- **qmd** ([github.com/tobi/qmd](https://github.com/tobi/qmd)) — опциональный локальный поисковый движок по markdown (гибридный BM25/vector + LLM re-ranking, с CLI и MCP-сервером) для роста за пределы того, что тянет index.md.
- **index.md хорошо работает примерно до ~100 источников, ~сотен страниц** — не нужна embedding-инфраструктура до этого порога. Полезная опорная точка для нашего масштаба (сейчас 14 источников).
- **Скачивание изображений локально**: в Obsidian Settings → Files and links → Attachment folder path на `raw/assets/`, плюс горячая клавиша "Download attachments for current file". У нас `raw/assets/` создана, но не настроена и пуста — реальный пробел.
- **git-репозиторий** — вики можно просто положить под git ради истории версий, веток и совместной работы. Мы этого не делаем (проект не является git-репозиторием).
- **Разные контексты применения**: personal (цели, здоровье, психология), research, чтение книги (аналог фан-вики вроде Tolkien Gateway), business/team (Slack-треды, транскрипты встреч), competitive analysis, due diligence, планирование поездок, курсовые заметки, hobby deep-dives.
- Автор сам предпочитает ингестить по одному источнику и участвовать в процессе, но допускает батч-загрузку с меньшим контролем — соответствует и нашей практике (первые 3 источника по одному, затем батч из 7+4).
- Связь с идеей Vannevar Bush Memex (1945) — курируемое персональное хранилище знаний с ассоциативными связями; часть, которую Буш не решил — кто делает поддержку — теперь решает LLM.
