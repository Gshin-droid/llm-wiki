## Саммари
Закрывает реальный пробел вики: механика Dynamic Workflows (скриптовая оркестрация десятков-сотен субагентов) существует в Claude Code с мая 2026 (Week 22 dev digest), упоминалась в предыдущих changelog-снапшотах вики только словом "dynamic workflows" мимоходом, но отдельной страницы не было — при том что это ровно тот механизм, которым устроен этот самый автономный прогон вики (движок сессии — Claude Code, но конкретно эта задача идёт как обычная сессия, не как workflow; сам факт существования инструмента важен для будущих задач вики, которые могут этого потребовать — например, параллельный ingest батча источников или полный lint-проход).

## Ключевые тезисы
- **Что это:** JavaScript-скрипт, который Claude пишет под задачу, а runtime исполняет в фоне отдельно от диалога — план оркестрации переносится в код, а не остаётся в контекстном окне Claude по ходу диалога.
- **Отличие от субагентов/skills/agent teams** — официальная таблица сравнения (см. raw): кто решает что запускать дальше, где живут промежуточные результаты, что переиспользуется, масштаб (десятки-сотни агентов за прогон против "горстки" у agent teams), поведение при прерывании (resumable в той же сессии, в отличие от рестарта хода у остальных трёх).
- **Примитивы**: `agent()` — один субагент с изолированным контекстом (можно задать JSON Schema для структурированного вывода вместо прозы); `pipeline()` — прогон по одному элементу списка без барьера между стадиями; `parallel()` — барьер-синхронизация, ждёт все агенты разом (использовать только когда стадии реально нужны результаты всех предыдущих сразу).
- **Quality-паттерны из коробки** — то, ради чего это выгоднее просто "больше агентов": adversarial review (независимые агенты пытаются опровергнуть находки друг друга), judge panel (черновик решения с нескольких углов + взвешивание), loop-until-dry (искать, пока раунды не перестанут находить новое).
- **Встроенный `/deep-research`** — ровно тот же механизм, что стоит за скиллом `deep-research`, уже доступным в этой сессии (см. список skills в system-reminder).
- **Ограничения runtime**: до 16 одновременных агентов, 1000 суммарно за прогон, нет пользовательского ввода посередине (кроме permission prompts), нет прямого доступа к файловой системе из самого скрипта — только через агентов.
- **Стоимость выше обычного диалога** — предупреждение "Large workflow" при >25 запланированных агентов или прогнозе >1.5M токенов; гайдлайн размера (`small`/`medium`/`large`/`unrestricted`) в `/config` — совет модели, не жёсткий лимит.
- **Опция сохранения** — удачный прогон можно сохранить как переиспользуемую команду (`.claude/workflows/` проектный или `~/.claude/workflows/` личный), которая появляется в `/` автокомплите.
