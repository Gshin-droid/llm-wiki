# MCP 2026-07-28 Specification Release Candidate

**Загружено:** 2026-07-09

## Саммари
Release candidate новой версии спецификации MCP (`2026-07-28`), которую авторы называют крупнейшей ревизией протокола с момента запуска. RC вышел 21 мая 2026, финальная спецификация запланирована на 28 июля 2026 (на момент загрузки источника ещё не вышла); изменения breaking, поэтому заложено 10-недельное окно валидации для мейнтейнеров SDK. Первая полноценная страница про сам протокол MCP в этой вики — раньше MCP упоминался только вскользь как "мост к внешним сервисам" в нескольких source-страницах, без разбора архитектуры и её эволюции.

## Ключевые тезисы
- **Протокол становится stateless.** Убран handshake `initialize`/`initialized` и заголовок `Mcp-Session-Id` — любой запрос может обслужить любой инстанс сервера. Клиентские метаданные едут в поле `_meta` на каждом запросе. Практическое следствие: MCP-серверы масштабируются за обычным round-robin балансировщиком, без sticky-сессий и общего session store. Состояние всё ещё можно вести — через паттерн "handle" (инструмент возвращает идентификатор, модель передаёт его обратно в следующих вызовах).
- **MCP Apps** — расширение для server-rendered интерактивных HTML-интерфейсов в песочнице iframe; UI-действия идут через тот же JSON-RPC и тот же аудиторский путь, что и обычные вызовы инструментов.
- **Tasks** — из экспериментальной core-фичи превратилась в расширение с редизайном под statelessness: сервер отвечает на `tools/call` хендлом задачи, клиент сам опрашивает прогресс (`tasks/get`/`tasks/update`/`tasks/cancel`); `tasks/list` убран.
- **Авторизация** приведена ближе к практике OAuth 2.0/OpenID Connect: обязательная валидация `iss` (RFC 9207) против mix-up атак, привязка credentials к выдавшему authorization-серверу и т.д.
- **Формальная политика депрекации**: Roots, Sampling, Logging депрекируются с гарантией минимум 12 месяцев до возможного удаления.
- Расширена поддержка JSON Schema 2020-12 в схемах инструментов (composition, `$ref`, conditionals).
