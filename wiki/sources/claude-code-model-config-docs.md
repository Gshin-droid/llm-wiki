## Саммари
Официальная документация о том, как в Claude Code выбирается и настраивается модель: алиасы (`sonnet`, `opus`, `haiku`, `fable`, `best`, `opusplan`), приоритет источников настройки, ограничение выбора для организаций, автоматические механизмы переключения модели. Загружено в ответ на вопрос пользователя: есть ли в Claude Code режим автоматического выбора модели по типу задачи — ответ да, и называется `opusplan`.

## Ключевые тезисы

### Алиасы моделей
- `sonnet` — повседневный кодинг; `opus` — сложное рассуждение; `haiku` — простые задачи; `fable` — самые тяжёлые/долгие автономные задачи (не default ни на одном тарифе, включается только вручную `/model fable`); `best` — Fable 5 где доступен, иначе последний Opus.
- `opusplan` — **единственный официальный task-based автопереключатель**: Opus в Plan Mode → Sonnet при выполнении. Включается вручную (`/model opusplan`), дальше переключение внутри сессии автоматическое. Opus тарифицируется только на фазе планирования (экономия).

### Приоритет настройки модели
1. `/model` в сессии (сохраняется как дефолт с v2.1.153+)
2. `--model` при старте
3. `ANTHROPIC_MODEL` env var
4. `model` в settings-файле

### Три независимых автоматических механизма (важно не путать с `opusplan`)
1. **Fallback-цепочки** (`fallbackModel` в settings / `--fallback-model`) — при перегрузке/недоступности модели сервер сам пробует следующую в списке; действует только на текущий ход (turn), не на всю сессию. Не связано с типом задачи — чистая отказоустойчивость.
2. **Content-based automatic fallback (только Fable 5)** — классификаторы безопасности для кибербезопасности/биологии; при срабатывании запрос повторяется на Opus 4.8 (или 4.7 на Claude Platform on AWS), сессия остаётся на Opus до ручного `/model fable`. Можно переключить на режим подтверждения ("switch models when a message is flagged" в `/config`).
3. **Max-плана усечение по лимиту** — по данным Help Center (support.claude.com, не сам этот источник), при исчерпании лимита Opus сессия может откатиться на Sonnet автоматически; числовой порог не публикуется официально.

### Effort levels
`low/medium/high/xhigh/max` + режим `ultracode` (`/effort ultracode` — xhigh + динамические workflow-оркестрации для существенных задач, только в текущей сессии). Дефолт `high` на большинстве моделей (Opus 4.7 — `xhigh`). Организации могут капать максимальный effort по ролям.

### Restrict model selection (`availableModels`)
Для организаций — allowlist моделей, применяется везде, где можно выбрать модель (включая fallback-цепочки, opusplan-апгрейд, effort). Отдельный механизм — Organization model restrictions через admin console (Claude Enterprise).
