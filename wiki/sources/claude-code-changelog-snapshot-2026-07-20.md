## Ключевой тезис

**Claude Code больше не запускает скиллы `/verify` и `/code-review` автоматически** — раньше Claude мог сам решить подключить эти два скилла, когда считал их релевантными задаче (общий механизм автоматического подключения скиллов, описанный в [[claude-skills]]); теперь для обоих нужен явный вызов командой `/verify` или `/code-review`. Это не отключает механизм авто-подключения скиллов в целом — точечное исключение именно для этих двух системных скиллов.

Практическое следствие: если раньше можно было полагаться, что Claude сам догадается прогнать `/code-review` перед коммитом нетривиального изменения или `/verify` после правки продуктового кода, теперь это нужно просить явно.
