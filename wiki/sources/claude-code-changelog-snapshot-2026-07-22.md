## Ключевой тезис: `sandbox.filesystem.disabled` — избирательное отключение файловой изоляции сэндбокса (2.1.216)

Сэндбокс Bash-инструмента Claude Code (`/sandbox`, `sandbox.*` в `settings.json`) состоит из двух независимых слоёв — файловой и сетевой изоляции. Новая настройка `sandbox.filesystem.disabled: true` отключает **только** файловый слой, оставляя сетевую изоляцию (allowlist доменов) в силе. Назначение: тулчейны вроде `kubectl`/`terraform`, которым нужен широкий доступ к файловой системе, но не нужен произвольный сетевой доступ — раньше единственным обходом было `excludedCommands`, которое снимает обе изоляции разом.

Официальная документация прямо формулирует компромисс: при отключённой файловой изоляции и авто-разрешённых командах сэндбоксированная команда может записать файлы (shell rc-файлы, исполняемые в `$PATH`, даже `~/.claude/settings.json` — если бы не отдельная защита настроек), которые прочитает или выполнит более поздняя команда, и так расширить собственный доступ на следующий прогон. Явная рекомендация — включать только для доверенных нагрузок. Настройку можно включить исключительно из user/managed settings или флага `--settings`; из `.claude/settings.json`/`.claude/settings.local.json` проекта — нельзя, то есть склонированный чужой проект не может сам себе ослабить изоляцию.

Прямое применение принципа минимальных привилегий из [[ai-security-by-design]] — новый инструмент для более точечной настройки того, где именно проходит граница доверия, а не бинарный выбор "сэндбокс целиком / выключить целиком".

## Остальные два пункта (2.1.217, менее значимые)

- **Настраиваемые лимиты параллелизма и глубины вложенности субагентов.** Changelog формулирует одной строкой ("Implemented concurrency limits for subagents with configurable depth nesting"), без названий конкретных ключей настроек. Допроверено (2026-07-22, ежедневное закрытие пробелов): официальная страница `code.claude.com/docs/en/sub-agents` на момент проверки ещё не отражает эту строку — прямо утверждает "The limit is fixed and not configurable" про существующий лимит "до 5 уровней" ([[claude-code]], снапшот 2026-07-09), а `code.claude.com/docs/en/env-vars` не содержит переменной под concurrency/depth. Реальная нестыковка между только что вышедшим changelog и не успевшей его догнать документацией — не ошибка вики, детали не домысливаются, вопрос остаётся открытым до обновления доков.
- **Исправлено применение `--max-budget-usd` для фоновых субагентов** — баг-фикс: раньше бюджетный лимит на фоновые субагенты (`claude agents`) мог не соблюдаться корректно, теперь соблюдается.
