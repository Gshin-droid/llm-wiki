Главная находка прогона — **Artifacts теперь могут вызывать MCP-коннекторы зрителя при каждом открытии страницы**, то есть именно тот механизм, который описан в скилле `artifact-capabilities`, доступном в сессиях этой самой вики (`capabilities: {mcp: ...}` в инструменте Artifact) — раньше в вики не было отдельной документированной страницы на эту тему, только упоминание скилла в системном промпте.

## Значимые тезисы

- **Artifacts вызывают MCP-коннекторы зрителя (web)** — опубликованная страница-артефакт может при каждом открытии обращаться к MCP-коннекторам через собственные подключения того, кто её смотрит (не автора), так дашборд показывает живые данные и может выполнять действия по запросу, а не статичный снапшот на момент сборки. Зритель одобряет доступ перед первым вызовом коннектора на странице. Одновременно добавлены публичные ссылки на шаринг, роли редактора для совместного редактирования (Team/Enterprise) и артефакты, создаваемые из сессий Claude Tag (Claude в Slack).
- **Screen reader mode (CLI)** — полностью текстовый линейный вывод терминала вместо визуального интерфейса (боксы, спиннеры, перерисовка на месте), для VoiceOver/NVDA и т.п. Включается флагом `--ax-screen-reader`, переменной `CLAUDE_AX_SCREEN_READER` или настройкой `axScreenReader`.
- **`/fork` теперь создаёт полноценную фоновую сессию** (2.1.212) со своей строкой в `claude agents`, а не подсессию внутри текущего диалога — старое поведение (подсессия) переехало в отдельную команду `/subtask`.
- **Лимиты на runaway-циклы** (2.1.212): WebSearch-вызовы и порождение субагентов ограничены по умолчанию 200 за сессию (`CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION`, `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`) — защита от автономных сессий, зациклившихся на поиске или делегировании. Прямое продолжение темы безопасности автономной работы из [[ai-security-by-design]].
- **`EndConversation`-инструмент** (2.1.214) — Claude Code теперь может сам завершить сессию при явно оскорбительном поведении пользователя или попытках джейлбрейка, аналогично тому, что уже действует на claude.ai с 2025 года.
- **Серия фиксов bypass-уязвимостей в проверке permissions** (2.1.214) — несколько независимых дыр закрыты одним релизом: обход на Windows PowerShell 5.1, неверный разбор file-descriptor redirect в Bash-анализаторе прав (теперь fail closed вместо fail open), команды длиннее 10 000 символов всегда требуют подтверждения (раньше могли молча выполниться), zsh-модификаторы переменных внутри `[[ ]]` больше не считаются инертным текстом, `help`/`man`-команды с опасными опциями/сабституциями больше не авто-одобряются. Прямое подтверждение принципа [[ai-security-by-design|эшелонированной защиты]] — единственная функция (permission analyzer) регулярно находит новые обходы, что подтверждает: не полагаться на один механизм.
- **`Edit(src/**)`-правило чинится** (2.1.214) — баг, при котором однослойный `dir/**` allow-паттерн ошибочно разрешал запись в `dir/` на любой глубине дерева вместо только `<cwd>/dir`; практический вывод для правил в `.claude/settings.json` этой вики (см. [[claude-code]], раздел Permissions) — стоит перепроверить свои `allow`-паттерны на этот класс ошибки при следующем апдейте.
- **MCP-вызовы длиннее 2 минут уходят в фон автоматически** — сессия не блокируется долгим инструментом; порог настраивается `CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS`.
- Более мелкое: `claude auto-mode reset` сбрасывает конфиг auto mode; auto mode на Bedrock/GCP Agent Platform/Foundry больше не требует явного opt-in переменной; "always allow"-правила теперь сохраняются в корне репозитория (переживают переключение между worktree).
