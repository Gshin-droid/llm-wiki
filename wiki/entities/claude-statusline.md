# claude-statusline

**Тип:** инструмент (open-source, statusline-скрипт для Claude Code CLI)
**Актуально на:** 2026-07-08

## Что это
Минималистичный статуслайн для [[claude-code]] — показывает лимиты использования (токены/rate limit), текущую директорию и git-ветку прямо в статус-строке CLI. Устанавливается одной командой `npx @kamranahmedse/claude-statusline`: скрипт бэкапит старый статуслайн (если был), копирует новый в `~/.claude/statusline.sh` и сам прописывает `statusLine` в `~/.claude/settings.json`. Требует `jq`, `curl`, `git`.

Это **не расширение VS Code** — работает как нативная фича Claude Code CLI (поле `statusLine` в `settings.json`), независимо от редактора.

- Репозиторий: [github.com/nilbuild/claude-statusline](https://github.com/nilbuild/claude-statusline), автор `nilbuild`, MIT, ⭐ ~1324 (на 2026-07-08)
- npm-пакет для установки — `@kamranahmedse/claude-statusline` (другой namespace, чем автор репозитория на GitHub — расхождение видно прямо в README, не ошибка поиска)

## Проверка безопасности (2026-07-08)
По запросу пользователя разобран код перед установкой — скрытых закладок не найдено.

**Метод:** файлы получены напрямую (raw GitHub + npm registry tarball), прочитаны как текст (без исполнения), проанализированы вручную — не через ИИ-суммаризацию, чтобы исключить влияние возможных prompt-инъекций в самом коде на анализ.

**`bin/install.js`** — чисто. Только `fs`/`path`/`os`, копирует статичный скрипт в `~/.claude/statusline.sh`, добавляет `statusLine` в `settings.json`. Сетевых вызовов, `eval`, обфускации нет.

**`bin/statusline.sh`** — чисто, с одним местом, которое стоит понимать явно:
- Получает JSON через stdin — штатный контракт хука `statusLine` в Claude Code.
- Fallback, если Claude Code не передал лимиты через stdin: скрипт достаёт OAuth-токен из тех же мест, где его хранит сам Claude Code (`CLAUDE_CODE_OAUTH_TOKEN`, macOS Keychain `"Claude Code-credentials"`, `~/.claude/.credentials.json`, Linux `secret-tool`).
- Токен уходит ровно на один адрес — `https://api.anthropic.com/api/oauth/usage` (официальный домен Anthropic, проверен буквально) — и больше никуда. Ответ (сами цифры usage, не токен) кэшируется 60 сек в `/tmp/claude/statusline-usage-cache.json` — минорно: читаемо другими локальными юзерами на многопользовательской машине, но не секрет.
- Git-команды read-only, ограничены `$cwd`. Чтения посторонних файлов (`.env`, SSH-ключи и т.п.) нет.

**Supply-chain проверка:** установка идёт через `npx @kamranahmedse/claude-statusline`, то есть выполняется npm-пакет, а не код напрямую из репозитория `nilbuild` — в принципе разные вещи. Скачан реальный tarball версии 1.0.6 с `registry.npmjs.org` и сверен побайтово (SHA256) с файлами из GitHub — **хэши совпали полностью**, подмены/расхождения нет.

**Остаточная нестыковка (не угроза, не выяснено):** пакет на npm опубликован под именем `kamranahmedse` (известный разработчик, автор roadmap.sh), у пакета в реестре пустые поля `repository`/`homepage` — формально не связывает его с `nilbuild`. Раз содержимое побайтово совпадает, признаков malicious fork нет, но 100%-ного подтверждения "один и тот же автор" тоже нет.

**Вывод:** закладок, эксфильтрации на сторонние домены, обфускации или подмены npm-версии не обнаружено. Единственное обращение к чувствительным данным (OAuth-токен) идёт только на официальный API Anthropic за заявленной функцией — данными об лимитах.

## Установка (2026-07-08)

Установлен на машине пользователя (Windows). Нюансы:
- Установщик проверяет зависимости (`jq`, `curl`, `git`) через Unix-style проверку (`which`) — в нативной PowerShell это ложно фейлится на все три тула сразу (даже на `curl`/`git`, которые физически есть в PATH), с намёком `brew install jq` в ошибке. Нужно запускать через POSIX-shell (Git Bash), где `which` работает.
- `jq` на Windows отдельно не был установлен — поставлен через `winget install jqlang.jq`. WinGet кладёt бинарник как reparse-point в `%LOCALAPPDATA%\Microsoft\WinGet\Links`, который добавляется в **persistent user PATH** автоматически, но подхватывается только новыми процессами/сессиями — уже открытые терминалы его не видят, пока их не перезапустить.
- Результат: скрипт в `~/.claude/statusline.sh`, `statusLine` в `settings.json` → `bash "$HOME/.claude/statusline.sh"`. Требует перезапуска сессии Claude Code, чтобы конфиг подхватился (см. [[claude-code]], раздел про перечитывание `settings.json` только при старте сессии).
