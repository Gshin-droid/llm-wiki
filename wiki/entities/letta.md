# Letta (бывший MemGPT)

**Тип:** инструмент (memory-as-OS framework)
**Актуально на:** 2026-07-07

## Что это
Память по метафоре операционной системы: main context = RAM (жёсткий лимит, обычно 16-32K токенов), recall storage = swap (полный лог сообщений), archival storage = disk (агент сам решает, что туда выгружать через свои function calls). Ключевое отличие от Mem0/Zep: управление памятью — не отдельный сервис, а операция, которой владеет сам агент.

## Ограничения
Крутая кривая обучения (недели тюнинга системного промпта), лишние LLM-вызовы на каждое решение "что архивировать", мало production-историй по сравнению с конкурентами.

## Когда уместен
Нужен полный контроль над policy памяти (например: "помни жалобы 2 года, забывай транзакции через 30 дней"), регулируемые домены с требованием аудита.
